# 🎬 MoviePilot-Plugins

MoviePilot第三方插件库，提供了一系列实用的插件来增强MoviePilot的功能。

> ⚠️ 注意：本插件库为个人维护，代码结构参考了其他开源项目。推荐优先使用[官方插件库](https://github.com/jxxghp/MoviePilot-Plugins)。

## 📦 插件列表

1. MessageRouter
2. 名称：插件消息重定向
3. 功能：允许您根据插件名称或通知标题关键字，将特定的消息修改为指定的通知类型，或者独立提取并推送到指定的企业微信通道
4. 版本：v2.0.0

## 使用说明
⚙️ 前置准备：MoviePilot 系统通知配置
要使用本插件的“企微直推”功能，您必须先在 MoviePilot 系统中配置好企业微信通道：

登录 MoviePilot，进入 设定 -> 通知（消息推送）。

点击新增通知渠道，选择 企业微信 (Wechat)。

填入您的企微配置（Corpid、Secret、Agentid 等）。

【关键步骤】：在配置页面最上方的 名称 一栏，给这个通道起一个好记的名字（例如：115通知、MP主应用、通知通道2）。

保存并测试确保该通道本身可用。

💡 提示： 您可以添加多个企微通知渠道，赋予不同的“名称”，以便在插件中实现分类路由。

📖 插件配置与语法说明
在安装并启用本插件后，进入插件配置页面，您会看到以下设置项：

1. 基础开关
启用高级路由与企微直推：插件的主开关。

直推后阻断系统默认广播：强烈建议开启。开启后，一旦某条消息被成功拦截并直推到企微，系统将不会再把它发送到您的 Telegram/微信 等默认全局渠道。

2. 高级消息路由映射规则（核心配置）
在文本框内，每行填写一条规则。语法格式严格遵循：
[插件名称或关键字] : [目标消息类型] : [系统微信通知名称]

📌 匹配优先级说明：
插件名精确匹配（最高优先级）：如果填写的是插件的真实名称（如 豆瓣同步、订阅提醒），只要该插件发信，就会立即命中。

标题/正文关键字匹配（兜底机制）：部分插件（如 115网盘STRM助手）在底层发信时是“匿名”的，系统无法获取其插件名。此时插件会自动扫描发出的通知标题或正文，只要包含您填写的关键字就会命中。

执行优先级：如果同时配置了“改类型”和“企微直推”，系统会优先保证拦截与企微直推的执行。

❓ 常见问题 (FAQ)
Q: 为什么我配置了 115网盘STRM助手::115通道，却无法拦截？
A: 因为该插件在底层代码中隐藏了自己的名字。请不要写插件全名，而是观察它发出的通知标题（例如：✅【115网盘】全量生成 STRM 文件完成喵！），从中提取一段关键字即可，比如配置为 115网盘::115通道。

Q: 开启了阻断，其他没配置规则的插件还能正常发通知吗？
A: 完全可以。本插件只对您在规则框里写明了的“插件名”或“关键字”进行拦截。未命中的消息会无损透传给 MoviePilot 原系统，正常发送到您的 Telegram 等渠道。

Q: 日志里为什么会有“命中规则 ➔ 已接管 (常规防线)”或“(底层模块)”？
A: 这是为了对付不同作者编写的插件。有的插件很乖（走常规防线），有的则会直接调用 MP 最深层的发信代码（底层模块）。不论它们走哪条路，插件的 3 层防护网都会将其精准拿下，日志只是提示您它是在哪一层被截获的。
